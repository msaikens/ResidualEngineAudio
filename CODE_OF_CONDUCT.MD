# Code of Conduct

## Purpose

Residual Voice Engine is an early-stage voice engine and Unity package project. The goal is to build a useful, reliable, and respectful technical community around the project.

Everyone participating in this project is expected to communicate professionally and constructively.

## Expected behavior

Participants are expected to:

* Be respectful and considerate.
* Give technical feedback without personal attacks.
* Assume good intent when possible.
* Ask clarifying questions before escalating disagreement.
* Keep discussions focused on the project.
* Respect maintainers’ time and project priorities.
* Report bugs with enough detail to reproduce the issue.
* Avoid derailing technical discussions.

## Unacceptable behavior

Unacceptable behavior includes:

* Harassment, threats, or intimidation.
* Personal insults or hostile comments.
* Discriminatory language or behavior.
* Sexualized comments or unwanted attention.
* Trolling, baiting, or repeated bad-faith arguments.
* Publishing private information without consent.
* Spamming issues, pull requests, or discussions.
* Abusing security reports for non-security complaints.

## Enforcement

The maintainer may take action when behavior disrupts the project or community.

Possible actions include:

* Clarifying expectations.
* Editing or deleting comments.
* Locking issues or discussions.
* Closing issues or pull requests.
* Temporarily or permanently blocking users from the repository.
* Reporting abuse to GitHub when appropriate.

Enforcement decisions are made at the maintainer’s discretion.

## Reporting conduct issues

For conduct concerns, contact the maintainer through an appropriate private channel if available.

Do not use public issues for sensitive personal complaints.

Security vulnerabilities should be reported according to `SECURITY.md`, not through public issues.

## Scope

This Code of Conduct applies to:

* GitHub issues
* Pull requests
* Discussions
* Review comments
* Release discussions
* Project-related communication spaces

It also applies when someone represents the project in public technical discussion.

## Maintainer note

This project is currently early-stage. The maintainer may prioritize technical stability, security, and release readiness over broad community process until the project matures.
